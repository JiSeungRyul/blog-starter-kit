# 기능 명세서: 에이전트 최적화 (GEO/AIEO/AEO)

**기능 브랜치**: `001-agent-optimization`
**작성일**: 2025-11-20
**상태**: 초안
**입력**: 사용자 설명: "이 블로그는 seo에는 최적화 되어있지만, geo/ aieo/ aeo 같이 이에전트용 최적화하는 작업이 아직 되어있지 않아. 최적화 해줬으면 좋겠어"

## 사용자 시나리오 및 테스트

### 사용자 스토리 1 - AI 검색 엔진 인덱싱 (우선순위: P1)

콘텐츠 제작자로서, 내 블로그 게시물이 AI 검색 엔진(Perplexity, Bing Chat, Google SGE 등)에 의해 쉽게 파싱되어 내 콘텐츠를 인용한 정확한 답변을 제공할 수 있기를 원합니다.

**우선순위 선정 이유**: 이것이 핵심 요청 사항(GEO/AEO)입니다. AI 생성 답변에서의 노출을 높이는 것은 현대 트래픽에 매우 중요합니다.

**독립 테스트**: 구조화된 데이터 테스트 도구(Google Rich Results Test 등)를 사용하여 `Article` 스키마가 올바르게 구현되었으며 필요한 모든 필드(제목, 날짜, 작성자, 본문)를 포함하고 있는지 확인합니다.

**인수 시나리오**:

1.  **조건** 블로그 게시물 페이지가 주어졌을 때, **행동** 구조화된 데이터 도구로 검사하면, **결과** 오류가 없는 유효한 `Article` JSON-LD가 발견된다.
2.  **조건** 홈페이지가 주어졌을 때, **행동** 검사하면, **결과** 유효한 `WebSite` 또는 `Blog` JSON-LD가 발견된다.

### 사용자 스토리 2 - llms.txt를 통한 LLM 에이전트 접근 (우선순위: P2)

AI 에이전트(또는 이를 사용하는 개발자)로서, HTML을 크롤링하지 않고도 사이트 구조와 콘텐츠를 빠르게 이해할 수 있도록 표준화된 `/llms.txt` 파일에 접근하고 싶습니다.

**우선순위 선정 이유**: `llms.txt`는 에이전트가 콘텐츠를 효율적으로 소비할 수 있게 하는 "AIEO"의 떠오르는 표준입니다.

**독립 테스트**: 브라우저나 curl을 통해 `/llms.txt`로 이동합니다.

**인수 시나리오**:

1.  **조건** 사이트 루트가 주어졌을 때, **행동** `/llms.txt`를 요청하면, **결과** 사이트 제목, 요약, 최근 게시물 목록 및 링크가 포함된 텍스트 파일이 반환된다.
2.  **조건** `llms.txt` 파일이 주어졌을 때, **행동** 파싱하면, **결과** 제안된 표준 형식(Markdown 유사)을 따른다.

### 엣지 케이스

-   **빈 블로그**: 게시물이 없는 경우에도 `llms.txt`는 사이트 정보만으로 생성되어야 하며, 홈페이지의 JSON-LD는 여전히 유효해야 합니다.
-   **메타데이터 누락**: 게시물에 작성자나 이미지가 없는 경우, JSON-LD 생성은 스키마 유효성을 깨뜨리지 않고 기본값으로 대체하거나 필드를 생략해야 합니다.
-   **대용량 콘텐츠**: 게시물 수가 매우 많은 경우 `llms.txt`를 자르거나 페이지를 나눠야 할 수 있습니다(하지만 이 스타터 키트에서는 관리 가능한 크기로 가정합니다).

## 요구사항

### 기능 요구사항

-   **FR-001**: 시스템은 루트(`/public/llms.txt` 또는 생성된 경로)에서 `llms.txt` 파일을 제공해야 합니다.
-   **FR-002**: `llms.txt` 파일은 블로그 제목, 간단한 설명, 모든 블로그 게시물의 제목 및 URL 목록을 포함해야 합니다.
-   **FR-003**: 모든 블로그 게시물 페이지는 `Article` JSON-LD 구조화된 데이터를 포함해야 합니다.
-   **FR-004**: `Article` 스키마는 `headline`, `datePublished`, `dateModified`(가능한 경우), `author`, `description`, `image`를 포함해야 합니다.
-   **FR-005**: 홈페이지는 사이트 이름과 URL을 정의하는 `WebSite` JSON-LD 구조화된 데이터를 포함해야 합니다.
-   **FR-006**: 시스템은 `robots.txt`가 일반적인 AI 사용자 에이전트를 차단하지 않도록 보장해야 합니다(명시적으로 구성되지 않은 경우).

### 주요 엔티티

-   **LLMsTxt**: 사이트 콘텐츠의 텍스트/마크다운 표현.
-   **SchemaMarkup**: schema.org 사양을 따르는 JSON 객체.

## 성공 기준

### 측정 가능한 결과

-   **SC-001**: `llms.txt`가 200 OK 상태와 500ms 미만의 TTFB로 제공됩니다.
-   **SC-002**: Google Rich Results Test에서 Article에 대해 0개의 오류를 보고합니다.
-   **SC-003**: 사이트 구조가 AI 에이전트에게 투명하여 복잡한 HTML 파싱 없이 콘텐츠를 검색할 수 있습니다.

## 가정

-   기존 `_posts` 디렉토리가 진실 공급원(Source of Truth)입니다.
-   표준 `llms.txt` 형식(llmstxt.org 또는 유사한 커뮤니티 표준에서 제안)을 사용합니다.
-   새로운 외부 서비스는 필요하지 않습니다. 이는 순수하게 정적 파일 및 메타데이터 개선입니다.
